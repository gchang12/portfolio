General

Caching
First: How web-surfing works. When accessing a webpage, your browser must send in a request for what to download and stuff. The server then decides what to send to your browser. Assuming that the requested content is okay to send, the server, the master computer that responds to HTTP requests, sends that content your way. To economize on subsequent requests, your browser will save some of the requested content so that it doesn't need to redownload resources each and every time you ping the server for something.

HTML

Images
Are specified via <img> void tag
  • Does not require closing tag.
  • src attribute must be a path to a URL or an internal file
  • Size attributes protect against jank.
  • alt is required for screen-readers, and if the image is not found; short for 'alternative text'. Helps also with SEO and indexing (i.e. listing of webpage in search results)
  • Common file formats:
    • General: jpg, gif, png
    • Simple: gif, png
    • Photos: jpg
    Get images from open source resources, such as CreativeCommons
    Some photos require extra stipulations to be used in a webpage.
    Semantic Elements
    Describe meaning of content to machines and humans.
    Say something about type of content they contain
    Allow search engines to parse content better.
    Makes writing HTML easier.
    Are relatively new (some existed prior to HTML5).
    Not semantic: <div>, <span are for styling purposes.

    A goal of HTML5 is to segregate semantics and styling.
