Caching

First: How web-surfing works. When accessing a webpage, your browser must send in a request for what to download and stuff. The server then decides what to send to your browser. Assuming that the requested content is okay to send, the server, the master computer that responds to HTTP requests, sends that content your way. To economize on subsequent requests, your browser will save some of the requested content so that it doesn't need to redownload resources each and every time you ping the server for something.

CSS

Selectors

At an atomic level, one can select one of:
  • An element: html { ... }
  • A class: .myclass { ... }
  • An ID: #my-id { ... }
Pseudo-classes
One can select an element based on state.
Example: a:hover {...}
Pseudo-elements
Pseudo-elements
Also known as "pseudo-selectors"
style content based on its relation to DOM and external factors also
uses: when a user hovers over something; link's been visited;
One can select a specific part of an element.
Example: p::first-line {...}
Attribute selectors
One can select an element given that a specified attribute is set, possibly to a specific value.
Example: a[href] {...} selects anchor tags with set href attributes.
Example: a[href="google.com"] {...} selects anchor tags with href attributes set to 'google.com'.

Colors

Hexadecimal
Numbers of the form, #[0-9A-F]{6}, where A = 10, B = 11, etc.
Can represent colors.
Find the hex for a color via this W3 resource
HSL
Is supported in most browsers.
Hue is represented by a color wheel (0..360).
Saturation and lightness are represented by percentages.
RGB
A model in which the addition of colors are additive.
Each value for red, green, and blue is represented by an integer from 0 to 255.
Alpha
Represents opacity (transparency) of a color.
Specified as the fourth argument in the two color models above.

The Box Model

The box model describes the structure of every HTML element that's rendered. All elements are composed of four smaller rectangles: content, padding, border, and margin. Each of these is nested within the previous.

Think: Shipping box.

Center contains the content, the text or media.

Around the content goes the padding, which is like the bubble wrap.

Around the padding goes the border, which is like the box the content comes in.

Margin goes around the element as the space around the box.

Each layer of the box model can be configured via CSS.

CSS properties can be applied to

  • width
  • height
  • bg-color

It's good practice to set the width and height for a given element so that no flexibility is left for the content to take up the space it needs.

Padding does NOT obstruct the content at all.

Padding is invisible.

At least two rules are needed to make borders appear:

  • width
  • style

Shorthand takes argument as

  • width
  • style
  • color

Background color does not extend into the margin area of a box.

Declare margin via:

  • margin-top
  • margin-bottom
  • margin-left
  • margin-right

margin shorthand:

  • top, left-right, bottom
  • top-bottom, left-right
  • clock-style: top, right, bottom, left
  • all-sides

use box-sizing property to use alt-box model

  • set: 'border-box'

Otherwise, sizing will affect only the content box.

border defaults

  • width = 1px
  • style = none
  • color = same as content

Design mock-up

Mock-up image is an image created by a visual designer that indicates what they may want the web page to look like.

The mock-up is passed to developers for them to implement.

Match-up is rare.

Evaluating mock-up:

  • Evaluate HTML
  • Code HTML
  • Evaluate styles shown
  • Code CSS one style type at a time

Pseudo-selectors

  • enable users to style content based on relation to DOM, and many more.
  • e.g. mouse cursor, etc.
  • uses: style element when user hovers over it; visited and unvisited links.
  • denoted by colons.

Transitions

  • define how element will appear while transitioning between two states
  • first need to define different states' styles (e.g. hover, state, pseudo-classes)
  • four properties: property, duration, timing-function, delay
  • declare values for shorthand as follows: transition: <property> <duration> <timing-function> <delay>;
